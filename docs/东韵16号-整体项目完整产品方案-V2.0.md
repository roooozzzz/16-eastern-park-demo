# 东韵16号｜整体项目完整产品方案（含宠物经营系统）V2.0

> 文档目的：本方案覆盖“整个项目”，宠物经营系统是其中的战略特色模块，而非全部内容。

## 0. 项目总览

- 项目名称：东韵16号（16 EASTERN PARK）
- 项目定位：社区商业项目的数字化运营系统（Community OS）
- 空间定位：1公里覆盖半径，辐射约8万人
- 业务目标：提升人流粘性、商户营收和项目品牌势能

## 1. 理念与战略

### 1.1 核心理念
1. 从“物业管理工具”升级为“社区经营系统”
2. 从“一次性活动”升级为“可复现的经营机制”
3. 从“单点折扣”升级为“内容+权益+联名+社群”的复合运营

### 1.2 战略主线（项目级）
主线：**内容日常化 → 到店行为化 → 交易权益化 → 关系资产化**

- 内容日常化：每日瞬间、街区内容更新，形成日常打开理由
- 到店行为化：签到、任务、活动参与，形成稳定行为习惯
- 交易权益化：联名券、场景权益、会员积分，形成可衡量交易
- 关系资产化：用户画像、商户协同、品牌心智沉淀

## 2. 项目产品全景（前台+商户+后台）

## 2.1 用户前台（C端）
### 核心模块
1. 首页
   - 每日瞬间（内容流）
   - 社区活动（活动卡片+详情）
   - 签到任务（积分发放）
2. 商户页
   - 商户列表与分类筛选
   - 楼幢/楼层信息展示（1/2/3/9/10栋；1F/2F/3F/4F/B1/B2）
   - 宠物友好标识筛选
3. 权益页
   - 场景化权益（雨天借伞、快递代收等）
   - 联名互助权益（A店触发，B店核销）
4. 我的
   - 会员码、积分、消费记录
   - 宠物名片（宠物ID）

## 2.2 商户后台（Merchant）
1. 门店基础信息维护（名称、分类、楼幢、楼层、宠物友好）
2. 活动与优惠配置（标准模板化）
3. 券核销与台账管理
4. 经营数据查看（到店、核销、复访）

## 2.3 运营后台（Admin）
1. 商户管理
2. 内容管理（每日瞬间/Banner/活动）
3. 会员与积分管理
4. 联名规则中心
5. 宠物模板中心（欢迎礼/复购礼/生日礼）
6. 经营看板（周/月）

## 3. 项目特色体系（不仅宠物）

## 3.1 特色A：内容日常化运营
- 目标：提升日活与心智占领
- 机制：低成本高频内容发布（物业主导）
- 指标：内容到达率、阅读率、互动率

## 3.2 特色B：商户联名互助网络
- 目标：打通跨商户流量与交易
- 机制：A店触发、B店核销、规则可配置
- 指标：联名券发放量、核销率、客单提升

## 3.3 特色C：场景化权益系统
- 目标：从“折扣心智”转向“社区服务心智”
- 机制：雨天/快递/应急等场景权益，提升用户真实依赖
- 指标：权益使用率、复访率、满意度

## 3.4 特色D：宠物经营系统（重点子系统）
- 定义：宠物友好升级为宠物经营系统（不是一次性活动）
- 闭环：宠物身份 → 高频遛宠行为 → 联名交易转化 → 商户复购增收
- 价值：情感链接增强、到店频次提升、跨业态消费链延展

## 4. 宠物经营系统（作为全项目特色模块）

## 4.1 功能主线（轻量版本）
1. 宠物会员ID（宠物档案+主人绑定+二维码）
2. 每日遛宠打卡（连续激励）
3. 宠物联名券（跨店触发与核销）
4. 关怀权益（饮水点、拾便袋、应急协助）

## 4.2 两条红线（必须避免）
1. 只做内容，不做交易
2. 只做活动，不做制度

## 4.3 特色成立判据（满足任意2条）
1. 宠物相关用户月活占比 ≥ 25%
2. 宠物联名券核销率 ≥ 18%
3. 参与宠物联动商户客单提升 ≥ 8%

## 5. 数据与技术架构（最小可行）

## 5.1 数据结构（项目级）
- users（用户）
- merchants（商户：含楼幢/楼层/宠物友好）
- points_logs（积分流水）
- moments（内容）
- activities（活动）
- coupon_templates（券模板）
- coupon_issues（发券）
- coupon_redemptions（核销）
- joint_rules（联名规则）

## 5.2 宠物扩展表
- pets（宠物档案）
- pet_checkins（遛宠打卡）
- merchant_pet_programs（宠物项目参与关系）

## 5.3 技术建议
- 前端：H5（当前已跑通Demo）
- 后端：Nest.js + MySQL + Redis（后续阶段）
- 部署：GitHub + Vercel（前端）；Railway（数据库/后端）

## 6. 经营机制与商业化

## 6.1 商户标准化运营商品（即插即用）
1. 欢迎礼（首到转化）
2. 复购礼（7天2次）
3. 生日礼（传播裂变）

## 6.2 平台侧收益路径
1. 商户数字化服务费（模板、活动、数据服务）
2. 联名活动执行服务费
3. 品牌联合活动与赞助

## 7. 项目级分数文档（Scorecard）

总分100（项目总评分）：
1. 用户价值 35分
   - DAU/MAU、30天复访、内容互动
2. 商业价值 35分
   - 核销率、客单提升、商户复购
3. 运营效率 20分
   - 活动上线效率、模板复用率、人效
4. 品牌传播 10分
   - 社媒传播、UGC占比、口碑提及

### 宠物子评分（100分）
- 月活占比（30）
- 打卡连续率（20）
- 联名券核销率（25）
- 联动商户客单提升（25）

评级建议：
- A（85-100）：可规模复制，对外招商输出
- B（70-84）：机制有效，继续调参
- C（<70）：机制未闭环，不扩大投放

## 8. 实施路线图

### 阶段0（已完成）
- Demo功能跑通：内容、联名、场景权益、宠物基础展示
- 后台楼幢/楼层规则更新完成

### 阶段1（2周）
- 项目级规则面板上线（联名+模板）
- 宠物P0闭环：打卡、发券、核销、看板

### 阶段2（4-6周）
- 后端与数据库正式接入
- 周报看板和异常预警机制稳定运行

### 阶段3（8-12周）
- 扩大商户参与范围
- 形成可复制的招商话术和输出模板

## 9. 风险与控制

1. 运营复杂度失控：坚持模板化与标准化
2. 商户配合不足：降低接入门槛，先小范围验证
3. 指标不达预期：按周复盘，优先调规则不加复杂功能

## 10. 结论

东韵16号的完整方案应以“项目级经营系统”为主体，宠物经营系统作为特色引擎。只有把内容、交易、联名、场景和数据看板一起跑通，才能形成可持续、可传播、可复制的项目竞争力。
