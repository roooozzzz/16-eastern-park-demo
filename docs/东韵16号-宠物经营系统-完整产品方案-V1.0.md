# 东韵16号｜宠物经营系统完整产品方案（V1.0）

> 定位：将“宠物友好”升级为“宠物经营系统”，形成可复现、可量化、可传播的项目特色。

## 1. 理念与目标

## 1.1 核心理念
- 宠物不是装饰性功能，而是情感与交易之间的连接器。
- 目标不是做一次活动，而是形成稳定复现的营收机制。
- 原则：轻量、可运营、可复制，不走重 IoT 路线。

## 1.2 业务目标
- 用户侧：提升日活、周留存、30天复访。
- 商户侧：提升到店转化、客单与复购。
- 项目侧：形成“社区第三空间”差异化标签。

## 2. 设计思路与主线

主线闭环：**宠物身份 → 高频行为 → 跨店交易 → 商户复购增收**

- 情感入口：宠物名片、萌宠内容、社群互动
- 行为触发：每日遛宠打卡（轻交互）
- 交易承接：宠物联名券（A店触发/B店核销）
- 经营复盘：每周看板与阈值管理

## 3. 特色定义（可传播）

对外定义：**“东韵16号不是宠物友好商场，而是宠物主的社区第三空间。”**

成立条件（满足任意2条即可对外宣讲特色能力）：
1. 宠物相关用户月活占比 ≥ 25%
2. 宠物联名券核销率 ≥ 18%
3. 参与宠物联动商户客单提升 ≥ 8%（对照组）

## 4. 核心功能（P0/P1）

## 4.1 前台（用户端）
### P0（2周）
1. 宠物会员ID（宠物档案+主人绑定+二维码）
2. 每日遛宠打卡（每日1次，连续7天奖励）
3. 宠物联名券领取与核销状态展示
4. 宠物友好商户筛选与标签展示
5. 关怀权益专区（饮水点、拾便袋、应急协助）

### P1（后续）
1. 遛狗搭子匹配（同时间段/同路线）
2. 宠物生日月专题与社群挑战赛
3. UGC话题榜与商户联合内容机制

## 4.2 后台（运营/物业）
### P0
1. 模板中心：欢迎礼/复购礼/生日礼
2. 规则中心：触发条件、发券商户、核销商户、预算上限
3. 商户参与开关：是否参与、每日预算、核销时间窗
4. 周看板：核心3指标+异常预警（红黄绿）

### P1
1. 商户分层运营（头部/成长/观察）
2. 自动化策略建议（低核销自动调券）
3. 区域热力与时段投放建议

## 5. 商业机制（四两拨千斤）

标准化商品（商户即插即用）：
1. 欢迎礼：首次带宠到店礼（拉新）
2. 复购礼：7天2次到店赠品（复访）
3. 生日礼：宠物生日月权益（传播）

避免失败的两条红线：
1. 只做内容，不做交易（必须有核销）
2. 只做活动，不做制度（必须模板化可配置）

## 6. 数据与系统最小结构

建议最小数据表：
- `pets`（宠物档案）
- `pet_checkins`（打卡记录）
- `coupon_templates`（联名券模板）
- `coupon_issues`（发券记录）
- `coupon_redemptions`（核销记录）
- `merchant_pet_programs`（商户参与关系）

## 7. 里程碑与交付

- M1（第1周）：前台打卡+领券链路可跑通（Demo+后台开关）
- M2（第2周）：发券/核销/看板数据闭环可追踪
- M3（第3-4周）：优化券策略，验证客单与复访提升

## 8. 分数文档（Scorecard）

总分100，按经营结果评分：
1. 用户活跃（30分）
   - 宠物相关月活占比（15）
   - 连续打卡率（15）
2. 交易转化（40分）
   - 联名券核销率（20）
   - 联动商户客单提升（20）
3. 运营可持续（30分）
   - 商户参与率（10）
   - 模板复用率（10）
   - 活动上线效率（10）

评级建议：
- A（85-100）：可规模化复制并对外招商输出
- B（70-84）：机制有效，需优化策略参数
- C（<70）：说明机制未闭环，禁止扩大投放

## 9. 结论

该方案可成为项目特色，但必须坚持“经营系统化”，持续用周看板验证“活跃-核销-客单”三项核心结果，避免回到一次性活动路径。
